Spring Microservices

Author: Naresh Kumar Sanda

Service Discovery, API Gateway, Circuit Breakers, and Distributed Tracing.

1. Service Discovery (Eureka)

Microservices register their IP/Port with a central registry (Eureka Server) so they can discover each other without hardcoding URLs.

@EnableEurekaClient
@SpringBootApplication
public class OrderService {}

2. API Gateway

Acts as the single entry point for clients. Handles routing, authentication, and rate limiting.

spring: 
  cloud:
    gateway:
      routes:
        - id: user_route
          uri: lb://USER-SERVICE
          predicates:
            - Path=/api/users/**

3. Circuit Breaker (Resilience4j)

Prevents cascading failures. If Service B is down, Service A stops calling it immediately (Circuit OPEN) and returns a fallback response.
@CircuitBreaker(name = "inventoryService", fallbackMethod = "fallbackInventory")
public String getInventory() {
    return restTemplate.getForObject("http://INVENTORY-SERVICE/items", String.class);
}
