Auto-configuration, Actuator, Profiles, and production-ready defaults.

1. Auto-Configuration Magic

Spring Boot removes the need for complex XML configurations by guessing what you need based on the JARs on your classpath.

The @SpringBootApplication annotation is actually a combination of three annotations:

  • @Configuration: Marks the class as a source of bean definitions.
  • @ComponentScan: Tells Spring to scan the current package and sub-packages for Beans (@Service, @Component).
  • @EnableAutoConfiguration: The real magic. It triggers Spring Boot to look at the classpath and configure beans automatically. (e.g., If Tomcat is on the classpath, it sets up an embedded web server).

Conditionally Loading Beans

Auto-configuration heavily relies on @Conditional annotations.

@Configuration
// Only load this config if the 'DataSource' class is present on the classpath
@ConditionalOnClass(DataSource.class) 
// Only load if the user hasn't already defined a bean named 'dataSource'
@ConditionalOnMissingBean(name = "dataSource")
public class DataSourceAutoConfiguration {
    @Bean
    public DataSource dataSource() {
        return new HikariDataSource();
    }
}

2. Spring Boot Actuator

Actuator brings production-ready features to your application without writing code. It exposes REST endpoints to monitor your app.

# application.yml
management:
  endpoints:
    web:
      exposure:
        include: "health,info,metrics,prometheus"
  endpoint:
    health:
      show-details: always # Shows database/disk health details

Crucial Endpoints

  • /actuator/health: Used by Kubernetes for Liveness and Readiness probes.
  • /actuator/metrics: JVM memory, garbage collection pauses, HTTP request timings.
  • /actuator/prometheus: Exposes metrics in a format Prometheus can scrape.
  • /actuator/env: Shows all environment variables and properties loaded.

3. Profiles & Properties

Profiles allow you to segregate parts of your application configuration and make it only available in certain environments.

// Only load this Bean when the "dev" profile is active
@Component
@Profile("dev")
public class MockEmailService implements EmailService { ... }

Property Resolution Hierarchy (Highest priority wins)

  1. Command line arguments (--server.port=8080)
  2. OS Environment Variables (SERVER_PORT=8080)
  3. Profile-specific application properties (application-prod.yml)
  4. Standard application properties (application.yml)

4. Custom Starters

A "Starter" is simply a set of convenient dependency descriptors you include in your application. To build your own custom library that auto-configures itself when dropped into a Spring Boot project:

  1. Create your @Configuration class with @Conditional annotations.
  2. Create a file named META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports (Spring Boot 2.7+).
  3. Add the fully qualified name of your Configuration class to that file.
  4. When any project includes your JAR, Spring Boot will automatically discover and load your configuration!
