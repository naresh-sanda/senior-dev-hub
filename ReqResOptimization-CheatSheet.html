1. Offset vs Cursor Pagination

TypeSQL ExamplePros/Cons
Offset (Bad)LIMIT 10 OFFSET 1000000MySQL literally counts 1,000,000 rows just to throw them away. Extremely slow on deep pages.
Cursor/Keyset (Good)WHERE id > 1000000 LIMIT 10Uses the B-Tree index to instantly jump to the 1,000,000th record. Lightning fast.

2. Payload Compression (GZIP/Brotli)

JSON text compresses incredibly well (often 70-80% smaller). Enabling compression drastically reduces network transit time, especially on mobile.

# Spring Boot Application.yml
server:
  compression:
    enabled: true
    mime-types: application/json,text/html

3. HTTP Caching Headers (ETag)

The Problem: A client requests a 2MB JSON list of products every time they open the app, even though the products rarely change.

The ETag Solution: The server sends an ETag header (a hash of the data). Next time, the client sends If-None-Match: [hash]. The server checks the hash. If the data hasn't changed, the server returns 304 Not Modified with an empty body, saving 2MB of bandwidth!
