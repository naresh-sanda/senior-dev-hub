Hoisting, Closures, Event Loop, and Asynchronous JS.

1. Execution Context & Hoisting

Before executing code, the JS engine parses it and sets up memory (Creation Phase). Variables declared with var and function declarations are "hoisted" to the top of their scope.

The Temporal Dead Zone (TDZ): let and const are also hoisted, but they remain uninitialized. Accessing them before declaration throws a ReferenceError.
console.log(a); // Output: undefined (hoisted but uninitialized)
var a = 5;

hello(); // Output: "World!" (functions are fully hoisted)
function hello() { console.log("World!"); }

2. Lexical Scoping & Closures

A Closure is a function that remembers the variables in its outer lexical scope, even AFTER the outer function has finished executing. It is widely used for data privacy (encapsulation).
function createCounter() {
    let count = 0; // Private variable
    
    return function() { // This inner function forms a closure
        count++;
        console.log(count);
    };
}
const counter = createCounter(); // The outer function is done.
counter(); // Output: 1
counter(); // Output: 2 (It remembers 'count'!)

3. The Event Loop (Deep Dive)

JavaScript is strictly single-threaded (one Call Stack). It achieves concurrency using the Browser/Node Web APIs and the Event Loop.

  1. Call Stack: Executes synchronous code.
  2. Web API: Handles async tasks (Timers, HTTP requests) in the background.
  3. Microtask Queue: Holds Promise callbacks (.then()). High Priority!
  4. Macrotask Queue: Holds setTimeout callbacks. Low Priority!
console.log("1"); // Stack
setTimeout(() => console.log("2"), 0); // Macrotask
Promise.resolve().then(() => console.log("3")); // Microtask
console.log("4"); // Stack
// Output Order: 1, 4, 3, 2 (Microtasks run BEFORE Macrotasks)

4. The Prototype Chain

JS doesn't have classical Java inheritance. It uses Prototypal Inheritance. Every object has a hidden [[Prototype]] property pointing to another object.

When you access arr.map(), the engine looks at arr. If it doesn't find map, it looks up the chain to Array.prototype, where it finds the function.

5. Modern Async/Await

async/await is syntactic sugar over Promises, making asynchronous code look synchronous. It avoids "Callback Hell".

async function fetchUserTransactions(userId) {
    try {
        // Pauses execution of THIS function until Promise resolves
        const user = await fetch(`/api/users/${userId}`).then(res => res.json());
        const txns = await fetch(`/api/txns/${user.accountId}`).then(res => res.json());
        return txns;
    } catch (error) {
        console.error("Network failed", error);
    }
}
