1. JVM Memory Model

Heap Memory Shared among all threads. Stores all instantiated objects (e.g., new MyObject()). Subject to Garbage Collection. Divided into Young Gen, Old Gen.
Stack Memory Thread-specific. Stores local variables, method call frames, and object references (pointing to the Heap). Fast, LIFO, auto-cleared when methods exit.
Metaspace Replaced PermGen (Java 8+). Uses native memory to store class metadata, static variables, and constants.

2. Garbage Collection (GC)

GC automatically reclaims Heap memory used by objects that are no longer reachable.

  • G1GC (Garbage First): Default since Java 9. Divides the heap into regions and prioritizes regions with the most garbage. Balances throughput and latency.
  • ZGC / Shenandoah: Ultra-low latency collectors. Pause times do not increase with heap size (usually <1ms pauses).
  • GC Roots: The starting points for GC tracing (Local variables on the stack, static variables, active threads).

3. Advanced Streams API

List<Employee> employees = getEmployees();

// 1. Grouping and mapping
Map<String, List<String>> namesByDept = employees.stream()
    .collect(Collectors.groupingBy(
        Employee::getDepartment,
        Collectors.mapping(Employee::getName, Collectors.toList())
    ));

// 2. Partitioning (true/false)
Map<Boolean, List<Employee>> highlyPaid = employees.stream()
    .collect(Collectors.partitioningBy(e -> e.getSalary() > 100000));

// 3. FlatMap (Flattening nested lists)
List<String> allSkills = employees.stream()
    .flatMap(e -> e.getSkills().stream())
    .distinct()
    .collect(Collectors.toList());

4. CompletableFuture & Concurrency

Use CompletableFuture for non-blocking, asynchronous programming.

CompletableFuture<User> userFuture = CompletableFuture.supplyAsync(() -> fetchUser(id));
CompletableFuture<Order> orderFuture = CompletableFuture.supplyAsync(() -> fetchOrder(id));

// Combine results of two independent futures
CompletableFuture<Response> combined = userFuture.thenCombine(orderFuture, (user, order) -> {
    return new Response(user, order);
});

// Exception handling
combined.exceptionally(ex -> {
    log.error("Failed to fetch data", ex);
    return new Response(null, null); // Return fallback
});

5. Java 21+ Virtual Threads (Project Loom)

Traditional OS threads are expensive. Virtual threads are lightweight threads managed by the JVM, allowing you to run millions of concurrent tasks using a simple "thread-per-request" model without callbacks or reactive frameworks.

// Start a Virtual Thread
Thread.startVirtualThread(() -> {
    System.out.println("Running in a virtual thread!");
});

// Using an ExecutorService for Virtual Threads
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    IntStream.range(0, 10_000).forEach(i -> {
        executor.submit(() -> {
            Thread.sleep(Duration.ofSeconds(1)); // Does NOT block an OS thread
            return i;
        });
    });
} // Blocks until all virtual threads finish
