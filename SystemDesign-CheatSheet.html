1. CAP Theorem

A distributed system can only provide TWO of the following three guarantees:
  • Consistency (C): Every read receives the most recent write.
  • Availability (A): Every request receives a response (even if stale).
  • Partition Tolerance (P): System continues to operate despite network failures.

(Note: Network partitions (P) are unavoidable, so you must choose between CP or AP).

2. Scaling Strategies

TypeDescription
Vertical (Scale Up)Adding more RAM/CPU to a single server. Limited ceiling, has single point of failure.
Horizontal (Scale Out)Adding more servers and putting a Load Balancer in front. Infinite scaling.

3. Database Sharding

Splitting a massive database table across multiple servers. Consistent Hashing is used to determine which shard a specific user's data belongs to, minimizing data movement when adding new shards.
