End-to-end automation from Git to Production.

1. CI vs CD

PhaseMeaningActions
CIContinuous IntegrationCode push -> Compile -> Run Unit Tests -> Build Docker Image -> Push to Registry.
CDContinuous DeploymentPull Docker Image -> Deploy to Staging -> Run E2E Tests -> Deploy to Prod.

2. GitHub Actions (Sample)

name: Java CI/CD
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - name: Set up JDK 17
      uses: actions/setup-java@v3
      with:
        java-version: '17'
    - name: Build with Maven
      run: mvn clean package
    - name: Build & Push Docker Image
      run: docker build -t myapp:latest .

3. Deployment Strategies

  • Rolling Update: Replace pods one by one. Zero downtime, but you have old and new code running simultaneously for a minute.
  • Blue/Green: Spin up a duplicate Prod environment (Green). Test it. Once perfect, instantly switch the router traffic from Blue to Green. Easy rollback.
  • Canary: Route 5% of user traffic to the new version. Monitor for errors. Gradually scale to 100%.
