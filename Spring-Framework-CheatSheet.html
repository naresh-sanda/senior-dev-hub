1. Inversion of Control (IoC)

IoC shifts the responsibility of creating and wiring objects from your code to the Spring Container.
ApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);
MyService service = context.getBean(MyService.class);

2. Bean Scopes

ScopeDescription
singleton (Default)One instance per Spring IoC container.
prototypeA new instance is created every time the bean is requested.
requestOne instance per HTTP request (Web only).

3. Aspect-Oriented Programming (AOP)

Used for cross-cutting concerns (Logging, Security, Transactions). Spring uses JDK Dynamic Proxies or CGLIB to wrap beans.

@Aspect
@Component
public class LoggingAspect {
    @Before("execution(* com.app.service.*.*(..))")
    public void logBefore() { System.out.println("Method called"); }
}
