MySQL Beginner Cheat Sheet

Author: Naresh Kumar Sanda

The definitive guide to learning MySQL, packed with sample data and real-world queries.

1. Database Fundamentals

MySQL is a Relational Database Management System (RDBMS). It stores data in Tables (which look like Excel spreadsheets). Tables are made up of Rows (records) and Columns (fields/attributes).
Concept Description
Primary Key (PK)A unique identifier for a row (e.g., User ID). No two rows can have the same PK.
Foreign Key (FK)A column that links to the Primary Key of another table, creating a relationship.
QueryA command (written in SQL) sent to the database to insert, read, or delete data.

1B. Essential MySQL Data Types

When you create a column in a table, you must tell MySQL what kind of data it will hold.

String (Text) Types

TypeUse Case
VARCHAR(255)Variable-length text (emails, names, passwords). Max length is 255 in this example.
TEXTLong-form text (blog posts, descriptions). Can hold huge amounts of text.
CHAR(2)Fixed-length text. Always takes exactly 2 characters (e.g., State codes like 'TX', 'CA').

Numeric Types

TypeUse Case
INTWhole numbers (IDs, age, quantities).
DECIMAL(10, 2)Exact monetary values. Means 10 digits total, 2 of which are after the decimal (e.g., 99999999.99).
FLOAT / DOUBLEApproximate numbers with decimals (scientific calculations). Do NOT use for money!
BOOLEANTrue/False values (MySQL actually saves this as TINYINT(1) behind the scenes).

Date & Time Types

TypeUse Case
DATEDate only (YYYY-MM-DD), e.g., '2026-05-20'.
DATETIMEDate and Time (YYYY-MM-DD HH:MI:SS).
TIMESTAMPTracks exactly when a record was created or updated. Automatically converts to UTC.

3. DDL (Data Definition Language)

Used to create and modify the structure of your database (Tables, Columns).

Creating the Database and Tables

-- Create and switch to the database
CREATE DATABASE ecommerce;
USE ecommerce;

-- Create users table
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL
);

-- Create products table
CREATE TABLE products (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(150) NOT NULL,
    price DECIMAL(10, 2) NOT NULL,
    category VARCHAR(50)
);

-- Create orders table with Foreign Keys
CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    product_id INT,
    quantity INT DEFAULT 1,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (product_id) REFERENCES products(id)
);

Modifying Tables

-- Add a new column to users
ALTER TABLE users ADD COLUMN created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP;

-- Delete a table entirely (Careful!)
DROP TABLE orders;

4. DML (Data Manipulation Language)

Used to Insert, Update, and Delete the actual rows in your tables.

Inserting Data

-- Insert a single user
INSERT INTO users (name, email) 
VALUES ('Naresh Kumar', 'naresh@example.com');

-- Insert multiple products at once
INSERT INTO products (title, price, category) VALUES 
('Laptop', 1200.00, 'Electronics'),
('Mouse', 25.00, 'Electronics'),
('Coffee Mug', 15.00, 'Kitchen');

Updating Data

Warning: Always use a WHERE clause when updating! If you forget it, you will update every single row in the table! 
-- Increase the price of the Mouse
UPDATE products 
SET price = 29.99 
WHERE title = 'Mouse';

Deleting Data

-- Delete Bob from the database
DELETE FROM users 
WHERE name = 'Bob Johnson';

5. DQL (Data Query Language) - Querying Data

Basic SELECT

-- Read everything from products table
SELECT * FROM products;

-- Read specific columns
SELECT title, price FROM products;

Filtering with WHERE

-- Exact match
SELECT * FROM users WHERE id = 1;

-- Greater than
SELECT * FROM products WHERE price > 50.00;

-- Multiple conditions (AND / OR)
SELECT * FROM products WHERE category = 'Electronics' AND price < 100.00;

-- Pattern matching (LIKE)
-- % means "any characters". This finds emails ending in @example.com
SELECT * FROM users WHERE email LIKE '%@example.com';

-- List matching (IN)
SELECT * FROM products WHERE category IN ('Kitchen', 'Books');

Sorting and Pagination

-- Sort products by price highest to lowest
SELECT * FROM products ORDER BY price DESC;

-- Get the top 5 most expensive products
SELECT * FROM products ORDER BY price DESC LIMIT 5;

6. Aggregations & Grouping

Used to calculate summaries of data, like totals, counts, and averages.

-- Count the total number of users
SELECT COUNT(*) FROM users;

-- Find the average product price
SELECT AVG(price) FROM products;

GROUP BY

Groups rows that have the same values into summary rows.

-- How many products are in each category?
SELECT category, COUNT(*) as product_count
FROM products
GROUP BY category;

-- Total revenue generated per user (assuming quantity * price)
SELECT user_id, SUM(quantity) as total_items_bought
FROM orders
GROUP BY user_id;

HAVING (Filtering Grouped Data)

You cannot use WHERE on aggregated functions like COUNT(). You must use HAVING.

-- Find categories that have more than 10 products
SELECT category, COUNT(*) 
FROM products
GROUP BY category
HAVING COUNT(*) > 10;

7. SQL Joins

Joins are used to combine rows from two or more tables, based on a related column between them (like Foreign Keys).

INNER JOIN

Returns records that have matching values in both tables. If a user hasn't made an order, they won't show up here.

SELECT users.name, orders.quantity
FROM users
INNER JOIN orders ON users.id = orders.user_id;

LEFT JOIN

Returns all records from the left table (users), and the matched records from the right table (orders). Users without orders will show NULL for their order quantity.

SELECT users.name, orders.quantity
FROM users
LEFT JOIN orders ON users.id = orders.user_id;

Multi-Table Join (The Real Power)

Let's join all three tables to see EXACTLY who bought what and for how much.

SELECT users.name, products.title, products.price, orders.quantity
FROM orders
INNER JOIN users ON orders.user_id = users.id
INNER JOIN products ON orders.product_id = products.id;

7B. Real-World Business Scenarios

Syntax is great, but combining tools to answer real questions is what makes a great developer. Let's solve actual business problems using our E-Commerce schema!

Scenario 1: "Who are our top 3 highest-spending customers?"

This requires joining all three tables to get the user name, product price, and order quantity, then calculating the total spend per user.

SELECT 
    users.name, 
    SUM(orders.quantity * products.price) AS total_spent
FROM users
INNER JOIN orders ON users.id = orders.user_id
INNER JOIN products ON orders.product_id = products.id
GROUP BY users.id, users.name
ORDER BY total_spent DESC
LIMIT 3;

Scenario 2: "Which products have NEVER been ordered?"

A classic technique to find "orphaned" data. We use a LEFT JOIN from products to orders. If a product has no orders, the order ID will be NULL.

SELECT products.title 
FROM products
LEFT JOIN orders ON products.id = orders.product_id
WHERE orders.id IS NULL;

Scenario 3: "Give me a Monthly Sales Report"

Working with dates is incredibly common. Here we use MySQL's built-in date functions to group revenue by the month the order was placed. (Assumes an order_date column in orders)

SELECT 
    YEAR(order_date) AS sales_year,
    MONTHNAME(order_date) AS sales_month,
    SUM(quantity * price) AS monthly_revenue
FROM orders
INNER JOIN products ON orders.product_id = products.id
GROUP BY sales_year, sales_month
ORDER BY sales_year DESC, MONTH(order_date) DESC;

8. Advanced Beginner Concepts

Subqueries

A query nested inside another query.

-- Find products that cost more than the average product
SELECT title, price 
FROM products 
WHERE price > (SELECT AVG(price) FROM products);

CTEs (Common Table Expressions)

A cleaner, more readable alternative to subqueries using the WITH keyword.

WITH ExpensiveProducts AS (
    SELECT id, title FROM products WHERE price > 1000
)
SELECT * FROM ExpensiveProducts;

9. DCL (Data Control Language)

Used to control access and manage permissions for users.

GRANT & REVOKE

-- Create a new user
CREATE USER 'readonly_user'@'localhost' IDENTIFIED BY 'password123';

-- Give them permission to ONLY read the ecommerce database
GRANT SELECT ON ecommerce.* TO 'readonly_user'@'localhost';

-- Remove a permission
REVOKE SELECT ON ecommerce.* FROM 'readonly_user'@'localhost';

10. TCL & Best Practices

Indexes

If you constantly search by email, querying a table with millions of rows is slow. Indexes build a "Table of Contents" for that column, making lookups blazing fast.

-- Create an index on the email column
CREATE INDEX idx_user_email ON users(email);

TCL (Transaction Control Language)

Groups multiple SQL statements into a single unit of work. If one statement fails, the entire transaction rolls back, preventing corrupt/partial data (like money leaving one account but never arriving in another).

START TRANSACTION;

UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;

-- If both succeeded:
COMMIT;

-- If something crashed/failed:
ROLLBACK;
