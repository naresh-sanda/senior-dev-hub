1. Big O Notation (Time/Space Complexity)

Big O describes how the runtime or space requirements grow as the input size (N) grows towards infinity. Drop constants and non-dominant terms (e.g., O(2N + N²) becomes O(N²)).
ComplexityNameData Structure / Algorithm Example
O(1)ConstantHashMap lookup, Array index access.
O(log n)LogarithmicBinary Search on sorted array, inserting into a BST.
O(n)LinearLooping an array, traversing a Linked List.
O(n log n)Log-LinearMerge Sort, Quick Sort (average), Java's Arrays.sort().
O(n²)QuadraticNested loops (Bubble Sort, Selection Sort).
O(2^n)ExponentialNaive Fibonacci recursive function.

2. Arrays & Strings (Two Pointers / Sliding Window)

Arrays and Strings are the most common interview topics. The key to optimizing O(N²) nested loops is using the Two Pointer or Sliding Window patterns.

Sliding Window (Find max sum of K consecutive elements)

public int maxSubArraySum(int[] arr, int k) {
    if (arr.length < k) return -1;
    int maxSum = 0, windowSum = 0;
    // 1. Calculate sum of first 'k' elements
    for (int i = 0; i < k; i++) {
        windowSum += arr[i];
    }
    maxSum = windowSum;
    // 2. Slide the window: subtract outgoing element, add incoming element
    for (int i = k; i < arr.length; i++) {
        windowSum += arr[i] - arr[i - k];
        maxSum = Math.max(maxSum, windowSum);
    }
    return maxSum; // O(N) Time, O(1) Space
}

3. Linked Lists (Fast & Slow Pointers)

Tortoise and Hare Algorithm: Use a Slow pointer (moves 1 step) and a Fast pointer (moves 2 steps) to detect cycles or find the middle element in a Linked List in O(N) time and O(1) space.
public boolean hasCycle(ListNode head) {
    if (head == null || head.next == null) return false;
    ListNode slow = head;
    ListNode fast = head.next;
    
    while (slow != fast) {
        if (fast == null || fast.next == null) return false; // Reached the end
        slow = slow.next;        // Moves 1 step
        fast = fast.next.next;   // Moves 2 steps
    }
    return true; // Fast pointer lapped the slow pointer (Cycle detected!)
}

4. HashMaps (Deep Internal Mechanics)

A HashMap provides O(1) average time complexity for inserts and lookups. It is backed by an array of Buckets.

  • Hashing: The key's hashCode() determines which array index (Bucket) it belongs to.
  • Collisions: If two different keys hash to the same bucket, they form a Linked List at that bucket (Separate Chaining).
  • Java 8 Optimization: If a bucket's Linked List exceeds 8 elements, Java converts it into a Red-Black Tree, improving worst-case search time from O(N) to O(log N)!
  • Load Factor: When the map is 75% full (0.75 default load factor), the array size is doubled (Rehashing).

5. Trees (Binary Search Trees & Traversals)

A BST is ordered: Left child < Node < Right child. Lookups are O(log N). Understanding DFS traversals via recursion is critical.

class TreeNode {
    int val;
    TreeNode left, right;
}

// In-Order Traversal (Left, Root, Right) -> Prints BST in ascending sorted order!
public void inorder(TreeNode root) {
    if (root == null) return;
    inorder(root.left);           // 1. Go Left
    System.out.println(root.val); // 2. Visit Node
    inorder(root.right);          // 3. Go Right
}

6. Graph Traversals (BFS vs DFS)

AlgorithmData StructureUse Case
BFS (Breadth-First)QueueFinding the shortest path in an unweighted graph. Explores neighbors level by level.
DFS (Depth-First)Stack / RecursionExploring all paths, solving mazes, topological sorting. Goes as deep as possible before backtracking.

BFS Implementation (Shortest Path)

public void bfs(Node start) {
    Queue queue = new LinkedList<>();
    Set visited = new HashSet<>();
    
    queue.add(start);
    visited.add(start);
    
    while (!queue.isEmpty()) {
        Node current = queue.poll();
        System.out.println(current.val);
        
        for (Node neighbor : current.neighbors) {
            if (!visited.contains(neighbor)) {
                visited.add(neighbor); // Mark visited BEFORE enqueuing
                queue.add(neighbor);
            }
        }
    }
}
