Docker Containerization

Author: Naresh Kumar Sanda

From basic containers to multi-stage builds and Docker Compose.

1. Core Concepts

Image A read-only template containing the application code, libraries, dependencies, and tools. (Think: Java Class)
Container A runnable, isolated instance of an image. (Think: Java Object)
Volume Persistent storage mechanism. Data inside a container is lost when it stops; volumes persist data safely outside the container lifecycle.
Registry A repository for Docker Images (e.g., Docker Hub, AWS ECR, GCP GCR).

2. Writing Dockerfiles

The Dockerfile is the recipe to build your image.

# Use a lightweight base image
FROM openjdk:17-jdk-slim

# Set metadata
LABEL maintainer="nsanda@example.com"
LABEL version="1.0"

# Create a non-root user for security
RUN groupadd -r appuser && useradd -r -g appuser appuser

# Set working directory
WORKDIR /app

# Copy dependency files first to leverage layer caching
COPY pom.xml mvnw ./
COPY .mvn .mvn/
RUN ./mvnw dependency:go-offline

# Copy the rest of the application
COPY src ./src

# Build the application
RUN ./mvnw package -DskipTests

# Run as non-root
USER appuser

# Expose port
EXPOSE 8080

# The command to run when container starts
ENTRYPOINT ["java", "-jar", "target/myapp.jar"]

3. Multi-Stage Builds

Multi-stage builds are crucial for creating small, secure production images by discarding build tools (like Maven/Gradle) and keeping only the final compiled artifact.

# --- Stage 1: Build ---
FROM maven:3.8-eclipse-temurin-17 AS builder
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

# --- Stage 2: Runtime ---
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app

# Only copy the built artifact from the builder stage
COPY --from=builder /app/target/myapp.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]

4. Networking & Volumes

Network Types

  • Bridge (default): Isolated network where containers can talk to each other via IP address.
  • Host: Removes network isolation between the container and Docker host (Linux only). Container uses host's network directly.
  • None: Container has no external networking.

Volumes

# Create a named volume
docker volume create pgdata

# Run PostgreSQL using the volume to persist database data
docker run -d --name mydb \
  -v pgdata:/var/lib/postgresql/data \
  -e POSTGRES_PASSWORD=secret postgres:14

5. Docker Compose

Define and run multi-container applications using a YAML file.

version: '3.8'

services:
  web:
    build: .
    ports:
      - "8080:8080"
    environment:
      - DB_HOST=db
      - DB_USER=postgres
    depends_on:
      - db
    networks:
      - app_network

  db:
    image: postgres:14
    environment:
      - POSTGRES_PASSWORD=secret
    volumes:
      - db_data:/var/lib/postgresql/data
    networks:
      - app_network

volumes:
  db_data:

networks:
  app_network:
    driver: bridge

6. Essential CLI Commands

# Building & Running
docker build -t myapp:1.0 .         # Build image in current dir
docker run -d -p 80:8080 myapp:1.0  # Run in detached mode (-d), map host port 80 to container 8080

# Inspection & Debugging
docker ps                           # List running containers (-a for all)
docker logs -f <container_id>       # Follow container logs
docker exec -it <id> /bin/sh        # Open a shell inside running container
docker inspect <id>                 # View low-level details (IP, volumes, etc.)

# Cleanup
docker rm -f <container_id>         # Force remove a container
docker rmi <image_id>               # Remove an image
docker system prune -a --volumes    # DANGER: Removes ALL unused containers, networks, images, and volumes
