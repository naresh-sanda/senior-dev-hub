Caching Strategies

Author: Naresh Kumar Sanda

Redis, Memcached, Write-through vs Write-back, and Eviction policies.

1. Redis vs Memcached

FeatureRedisMemcached
Data StructuresStrings, Hashes, Lists, SetsStrings only
PersistenceYes (RDB snapshots, AOF)No (In-memory only)
Use CaseLeaderboards, Pub/Sub, Complex CachingSimple HTML/Session caching

2. Caching Patterns

  • Cache Aside: App checks cache. If miss, reads from DB and updates cache. (Best for read-heavy).
  • Write-Through: App writes data to BOTH Cache and DB simultaneously. (Slower writes, fast reads).
  • Write-Back: App writes to Cache ONLY. Cache asynchronously writes to DB later. (Extremely fast, but risk of data loss on crash).

3. Cache Eviction (LRU/LFU)

LRU (Least Recently Used): Evicts items that haven't been accessed in the longest time.
LFU (Least Frequently Used): Evicts items that are accessed the least often over time.
