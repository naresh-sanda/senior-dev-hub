Spring Data JPA & JDBC

Author: Naresh Kumar Sanda

Entity mapping, Repositories, and the N+1 Query Problem.

1. Entity Mapping

@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, unique = true)
    private String email;

    @OneToMany(mappedBy = "user", cascade = CascadeType.ALL)
    private List orders;
}

2. Spring Data Repositories

Spring dynamically implements this interface at runtime. No SQL required!

public interface UserRepository extends JpaRepository {
    // Magic finder method
    List findByEmailContainingIgnoreCase(String email);
    
    // Custom JPQL Query
    @Query("SELECT u FROM User u WHERE u.status = 'ACTIVE'")
    List findActiveUsers();
}

3. The N+1 Query Problem

The Trap: You query 100 Users (1 query). Then you loop through them and call user.getOrders(). Because of Lazy Loading, Hibernate fires 100 extra queries to get the orders. (1 + 100 = 101 queries!).

The Solution: Use JOIN FETCH or EntityGraphs to grab the orders in the very first query.

@Query("SELECT u FROM User u JOIN FETCH u.orders")
List findAllUsersWithOrders();
