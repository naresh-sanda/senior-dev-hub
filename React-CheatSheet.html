Hooks, State Management, and Rendering Optimization.

1. How React Renders (The Virtual DOM)

React maintains a lightweight copy of the real DOM called the Virtual DOM. When state changes, React builds a NEW Virtual DOM, compares it to the old one (Reconciliation / Diffing), and updates only the specific real DOM nodes that changed. This makes UI updates incredibly fast.

2. The Big 3 Hooks (useState, useEffect, useContext)

useEffect: Manages side effects (API calls, subscriptions, manual DOM changes). The dependency array [] dictates when it runs.
import { useState, useEffect } from 'react';

const UserData = ({ userId }) => {
    const [user, setUser] = useState(null);

    useEffect(() => {
        // Runs on mount AND whenever 'userId' changes
        fetch(`/api/users/${userId}`)
            .then(res => res.json())
            .then(data => setUser(data));
            
        // Cleanup function (runs on unmount or before next effect)
        return () => { console.log("Cleaning up"); }; 
    }, [userId]); // Dependency Array
    
    if (!user) return <p>Loading...</p>;
    return <h1>{user.name}</h1>;
};

3. State Management (Context API vs Redux)

Prop Drilling Problem: Passing a 'theme' or 'user' prop down through 10 layers of components just so the bottom component can read it.

Context API Solution: Creates a global teleportation tube for data. The top component uses <Provider> to broadcast data, and any deeply nested component uses useContext() to grab it instantly.

Note: For extremely complex, high-frequency state changes, use Redux or Zustand instead of Context to prevent full app re-renders.

4. Performance Optimization (useMemo & useCallback)

When a parent component re-renders, ALL of its children re-render by default. If a child does heavy calculation, this is a massive performance hit.
  • React.memo(Component): Caches the component. It only re-renders if its Props change.
  • useMemo(() => computeValue(), [deps]): Caches the RESULT of an expensive calculation so it isn't recalculated on every render.
  • useCallback(fn, [deps]): Caches a FUNCTION reference. Useful when passing callback functions as props to React.memo children, preventing unnecessary re-renders.

5. Writing Custom Hooks

Custom hooks allow you to extract reusable logic (like fetching data or managing a form) into a standard JS function.

// Reusable Custom Hook
function useWindowWidth() {
    const [width, setWidth] = useState(window.innerWidth);
    
    useEffect(() => {
        const handleResize = () => setWidth(window.innerWidth);
        window.addEventListener('resize', handleResize);
        return () => window.removeEventListener('resize', handleResize);
    }, []);
    
    return width;
}

// Usage in Component
const MyComponent = () => {
    const width = useWindowWidth();
    return <p>Window is {width}px wide!</p>;
};
